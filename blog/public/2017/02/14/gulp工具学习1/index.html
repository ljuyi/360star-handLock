gulp工具学习1

2017-02-14 • 标签: 工具

使用gulp对项目进行构建时，我们会使用 gulp.src 接口将匹配到的文件转为 stream（流）的形式，再通过 .pipe() 接口对其进行链式加工处理。
对比流式读取文件，和块式读取文件的区别：
流式：源源不断地输送到输出接口，相当于一边读取一边输出
块式：先整个从输入接口读取完毕后，在将整个文件输送到输出接口
这样对比一看，文件越大，流式操作的优点就越明显

Stream的分类：
⑴ Readable Streams
⑵ Writable Streams
⑶ Transform Streams
⑷ Duplex Streams
⑸ Classic Streams

其中 Transform Streams 和 Duplex Streams 都属于即可读又可写的流，而最后一个 Classic Streams 是对 Node 古早版本上的 Stream 的一个统称

Readable Streams:可读流，通过pipe接口可以将数据传递给一个writable,transform或者duplex流

readableStream.pipe(dst)
常见的Readable Streams包括：

  1. 客户端上的HTTP responses
  2. 服务端上的HTTP requests
  3. fs read streams(上面流式操作的demo就是创建了一个这个)
  4. zlib streams
  5. crypto streams
  6. TCP sockets
  7. 子进程的stdout和stderr
  8. process.stdin
    虽然Readable Streams是可读流，但将其传入一个消耗对象之前，它都是可写的

这种方式在消耗数据前（就是这里的process.stdout），数据会一直在缓存中，可以用接口_read()解决这个问题,在_read()方法内进行流数据的push

writable Streams

可写流接口是对写入数据的目标的抽象：
src.pipe(writableStream)
