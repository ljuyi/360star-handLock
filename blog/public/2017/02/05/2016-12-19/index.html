window对象

BOM核心对象是window，表示浏览器的一个实例，它既是浏览器窗口的一个接口，又是ECMAScript规定的Global对象

全局作用域

全局作用域中声明的变量、函数都会变成window对象的属性和方法

在全局作用域中定义变量和函数会自动转变成全局对象的属性和方法，即可以通过window.访问，但定义全局变量与在window对象上直接定义属性还是有一些差别：

全局变量不能通过delete操作符删除，而直接在window对象上定义的属性可以
因为通过var语句添加的window属性有一个名为[[Configurable]]的特性，这个特性的值被设置为false，因此不可以通过delete操作符删除

窗口关系及框架

如果页面中包含框架，则每个框架都拥有自己的window对象，并且保存在frames集合中（类似数组可以通过下标访问）
top parent self
top 对象始终指向最高（最外）层的框架，也就是浏览器窗口
parent 对象始终指向当前框架的直接上层框架，在没有框架的情况下，parent一定等于top（此时他们都等于window）
self 始终指向window，引入self对象的目的是为了与top和parent对象对应起来

上面这三个对象都是window对象的属性，可以通过window.top,window.parent,window.self等形式访问

窗口位置

获取位置

IE Safari Opera Chrome:
screenLeft : 窗口相对于屏幕左边的位置
screenTop : 窗口相对于屏幕上边的位置

Firefox
screenX : 窗口相对于屏幕左边的位置
screenY : 窗口相对于屏幕上边的位置

跨浏览器取得窗口左边和上边的位置

在IE,Opera中，screenLeft和screenTop的取值不包括浏览器工具栏的高度，即当浏览器顶端与屏幕顶端重合，screenTop的取值为浏览器的工具栏高度；而在Chrome,Firefox,Safari中包括工具栏的高度，即与上面的相反
在Firefox,Safari,Chrome中，始终返回页面中每个框架的top.screenX,和top.screenY的值，而IE,Opera则会给出框架相对于屏幕边界的精确坐标值

改变位置

window.moveTo(); //参数分别为新位置的x和y坐标值
window.moveBy(); //参数分别为在水平和垂直方向上移动的像素值

窗口大小

获取窗口大小

IE9+ Firefox Safari Opera Chrome提供了四个属性
innerWidth innerHeight outerWidth outerHeight
在IE9+，Safari，Firefox中，outerWidth和outerHeight返回浏览器窗口本身的尺寸
在Opera中，这两个属性值表示页面视图容器的大小，而innerWidth和innerHeight则表示该容器中页面视图区的大小（减去边框宽度）
在Chrome中，outerWidth,outerHeight与innerWidth,innerHeight返回相同的值，即视口大小而非浏览器窗口大小

document.documentElement.clientWidth(clientHeight) 在IE,Firefix,Safari,Opera,Chrome中有效，其中IE6中必须是标准模式有效，混杂模式无效，混杂模式时需用 document.body.clientWidth(clientHeight)
跨浏览器获取页面信息

改变窗口大小

window.resizeTo(); //参数分别为浏览器窗口的新宽度和新高度
window.resizeBy(); //参数分别为新窗口与原窗口的宽度和高度之差

导航和打开窗口

window.open();
既可以导航到一个特定的URL，也可以打开一个新的浏览器窗口，接受四个参数：要加载的URL，窗口目标，一个特性字符串以及一个表示新页面是否取代浏览器历史记录中当前加载页面的布尔值
四个参数都为可选参数，若都不填，则新打开一个空白页
eg:window.open(‘http://www.wrox.com/','wroxWindow','height=400,width=400,top=10,left=10,resizable=yes‘);
若第二个参数是已有的窗口或框架名，就会在具有该名称的窗口或框架中加载第一个参数指定的URL，此时第三个参数被忽略
若第二个参数是不存在的，就会新创建一个窗口，该窗口的名称为该参数值，且窗口初始化为长为400px，宽为400px，距屏幕上边和左边分别为10px
window.close();
可以用于关闭窗口，仅用于通过window.open()方法打开的窗口，window可以替换为窗口名称

间歇调用和超时调用

setTimeout:超时调用，只调用一次
setInterval:间歇调用，调用多次
两个方法传入两个参数，分别是调用的函数，和时间间隔（毫秒为单位）
第一个参数也可以是字符串，相当于eval的参数，但是不建议这样做，因为传递字符串可能导致性能缺失
第二个参数表示等待多长时间的毫秒数，但是经过这段时间后不一定会执行这段代码，因为这两种方法是将任务在指定时间过后添加到任务队列中，如果添加的时候，任务队列中没有其他任务，则会立即执行，如果有其他任务，则要等到其他任务完成后再执行这段代码。
这两个方法会返回一个数值ID，是计划执行代码的唯一标识符，可以通过它来取消超时调用（clearTimeout(),clearInterval()）。

超时调用的代码都是在全局作用域中执行的，因此函数中this的值在非严格模式下指向window对象，在严格模式下是undefined

系统对话框

alert() : 只显示指定内容的对话框，和一个确定按钮的对话框


prompt ：在confirm基础上外加一个文本域
prompt("请输入您的名字","Bill Gates");

location对象

location对象提供了与当前窗口中加载的文档有关的信息，还提供一些导航功能，它将URL解析成独立的片段，让开发人员可以通过不同的属性访问这些片段，它既是window对象的属性，也是document对象的属性，即window.location和document.location引用的是同一个对象
location的属性：

属性名 例子 说明
hash “#contents” 返回URL中的hash（#号后跟零或多个字符），
如果URL中不包含散列，则返回空字符串
host “www.wrox.com:80” 返回服务器名称和端口号（如果有）
hostname “www.wrox.com” 返回不带端口号的服务器名称
href “http://www.wrox.com“ 返回当前加载页面的完整URL。而location对
象的toString()方法也返回这个值
pathname “/WileyCDA/“ 返回URL中的目录和（或）文件名
port “8080” 返回端口号，若URL中没有则返回空字符串
protocol “http:” 返回页面使用协议，通常为http或https
search “?q=javascript” 返回URL的查询字符串。以问号开头

navigator对象

识别客户端浏览器的事实标准
属性或方法：（通常用于检测显示网页的浏览器类型）
appCodeName:浏览器的名称
appMinorVersion:次版本信息
appName:完整的浏览器名称
appVersion:浏览器的版本。一般不与实际的浏览器版本对应
buildID:浏览器编译版本
cookieEnabled:表示cookie是否启用
cpuClass:客户端计算机中使用的CPU类型（*86,68K,Alpha,PPC,Other）
javaEnabled():表示当前浏览器中是否启用了java
language:浏览器的主语言
mimeTypes:在浏览器中注册的MIME类型数组
onLine:表示浏览器是否连接到了因特网
oscpu:客户端计算机的操作系统或使用的CPU
platform:浏览器所在的系统平台
plugins:浏览器中安装的插件信息的数组
preference():设置用户的首选项
product:产品名称
productSub:关于产品的次要信息
systemLanguage:操作系统的语言
userAgent:浏览器的用户代理字符串
userLanguage:浏览器的默认语言
userProfile:借以访问用户个人信息的对象
vendor:浏览器的品牌
vendorSub:有关供应商的次要信息

history对象

保存用户上网的历史记录
方法：

  1. history.go():使用go方法可以调转到指定页面，接受两种参数
    (1) 整数值
    history.go(-1); //后退一页
    history.go(1); //前进一页
    history.go(2); //前进两页
    (2) 字符串
    跳转到历史记录中包含该字符串的第一个位置
    history.go('aaa.com'); //跳转到最近的aaa.com页面

  2. history.back():后退一页

  3. history.forward():前进一页

属性：
history.length:保存历史记录的数量，所有向后和向前的数量。加载到第一个页面时该值为0.
